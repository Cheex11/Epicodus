Leap Year Notes

BDD is a method used to answer specific questions when programming. It helps the developer design the interface before iumplementation. It stands for behavior driven design and has three parts:

  • Describe
  • Implement
  • Clean Up
The BDD Process
  1. Choose one behavior to build.
  2. Write a spec.
  3. Make sure it fails
  4. Implement the behavior as simply as possible.
  5. Check that the spec passes.
  6. Check that all specs pass
  7. Refactor
